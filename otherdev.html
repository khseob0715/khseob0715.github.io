광성공구 재고관리 시스템

3학년 2학기 데이터베이스 과목 최종 개인 프로젝트

개발 동기

일단, 광성공구철물상사는 저희 부모님 가게입니다. 가게는 공장이 밀집된 지역에 위치해 있기 때문에 고객사가 상당히 많으며 다양한 종류의 제품을 보유하고 있습니다. 그렇기 때문에 사소한 문제들이 자주 발생하고 있습니다. 예를 들면 재고의 수량이 파악되지 않은 상태에서 제품을 새로 주문하거나 제품의 가격이 규격 혹은 고객사마다 모두 다르기 때문에 금전적인 문제가 발생하고 있습니다. 또한 월말이 되면 부모님께서는 고객사에게 청구서를 발송하기 위해서 매우 바빠지셔서 항상 마음이 불편합니다. 고로 저는 부모님의 업무를 종종 도와드리고 있으며 부모님께서 가게의 제품과 가격, 청구서 등을 엑셀을 통해서 관리하고 계시는 것을 알게 되었습니다. 따라서 이 자료를 이용해 관리 시스템을 만들어 드리면 부모님의 업무량이 상당히 줄어들 것이라 생각을 하였기 때문에 개발하고자 하였습니다.

요구 사항 분석

1) 공정 과정에서 사용되는 제품을 판매하기 때문에 제품별 규격을 정확하게 구분
2) 고객사별 가격이 다르기 때문에 제품명과 고객사를 통한 가격 검색 기능.
3) 제품 검색을 통한 재고의 유무, 규격별 가격, 고객사별 가격, 판매량 조회 기능.
4) 고객사별 월말 청구서 작성, 조회 기능.

* 설계 시 필요한 가정 및 요점 사항
- 제품, 고객사 목록은 가게 엑셀파일 기준.
- 제품은 명칭, 규격, 가격, 재고, 생산처의 정보를 가짐.
- 고객사는 회사명, 대표자명, 주소, 전화번호, 이메일의 정보를 가짐.
- 기존 재고 등록을 통한 현 재고 관리가 필요.
- 새 물건, 새 고객사 등록이 가능하도록 구현.
- 부모님이 사용하시기 편해야 함.

My Review

처음으로 많은 양의 엑셀파일을 갖고 데이터베이스 관리 시스템 만들어 보았던 프로젝트였습니다. 데이터베이스를 막 배우기 시작했던 저는, 당시 개념이 바로 잡히지 않아 데이터를 정규화하는 작업에 굉장한 시간을 투자하고 시도했었습니다. 매우 힘들고 프로젝트를 포기하고 싶었지만 수 많은 노력끝에 정규화 규칙에 맞게 잘 정리하였습니다. 그래도 이러한 과정을 거치면서 데이터를 정리하는 방법을 깨닫게 되었고, 시스템 구현을 훨씬 수월하게 진행하게되어 성공한 프로젝트로 기억하고 있습니다.

YOLO API Instruction

3학년 2학기 오픈소스 소프트웨어 최종 팀 프로젝트 OSS 설명서 작성


Yolo API란?

객체 탐지 기법에 있어서 새로운 접근법인 YOLO를 소개합니다.
기존 객체 탐지 기법은 사전 작업으로 탐지를 먼저 수행합니다. 하지만 YOLO API 시스템은 객체 탐지를 공간적으로 분리된 경계공간들과 class(분류할 object 종류들)에 대한 확률과 연관된 회귀문제로 재정의하였습니다. API의 단일 신경망 네트워크는 전체 이미지에서 한번의 평가를 통해서 직접적으로 경계공간들을 예측하고 class의 확률을 계산합니다. 또한 전체 탐지 파이프라인이 단일 네트워크이기 때문에, 탐지 성능이 최적화됩니다. YOLO API의 통합된 아키텍쳐는 매우 빠르게 수행이 되는데 45 FPS로 실시간 이미지 연산이 가능합니다. 또한 다른 버전인 Fast YOLO는 놀랍게도 다른 실시간 탐지모델보다 2배의 mAP(평균 정확도)를 가지면서 155 FPS의 성능을 보여줍니다. 최신 탐지시스템과 비교했을 때, YOLO는 localization error가 조금 더 높지만, 배경으로 인한 예측 실패율은 더 낮습니다. 마지막으로 YOLO API의 장점은 탐지의 아주 일반적인 표현들을 학습합니다. 이 말은 DPM이나 R-CNN같은 다른 탐지 대비 다른 영역에서의 자연스러운 이미지로부터 일반적인 표현을 학습하는데 있어서 결과가 더 좋다는 것을 의미합니다.


YOLO API를 선택한 이유

Computer vision 분야에 대한 관심이 많던 제가 예전부터 알고 있던 API였습니다. 하지만 당시 저는 Python으로 구현된 프로그램과 Github 사용법을 몰랐기 때문에 YOLO API가 어렵게만 느껴져 해당 API를 전혀 활용해보지 못했습니다. 그 기억이 아쉬움으로 남아 저는 저희 팀원들에게 이 YOLO API를 선택해 Python 프로그래밍과 OpenCV, Computer Vision 분야에 대해서 함께 공부해보자고 제안했었습니다.

그리고 무엇보다도 API의 이름이 너무 마음에 들었습니다.
요즘 라이프스타일 트렌드인 YOLO(You Only Live One)와 이름이 비슷한 You Only Look Once인 YOLO API!

팀 프로젝트를 팀원들 모두 재미있게 즐기면서 하자는 의미에서 선택을 하였습니다.

프로젝트의 자세한 내용은 Github 링크를 통해 확인해 주셨으면 좋겠습니다.!
