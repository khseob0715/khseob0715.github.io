Triplay

제 2회 소프트웨어 중심대학 오픈소스 해커톤

팀 주제: AR 게임기반 관광 알리미 서비스


집순이, 집돌이를 위해 여행 알림이 서비스 어플리케이션을 제공해 삶에 활력을 되찾아주자!
관광지에 AR 게임 요소를 더해 지역 축제를 성공으로 이끌자!

시스템 흐름도

안드로이드 기기의 어플은 실행과 동시에 그 날의 관광 정보를 공공 데이터 포털의 Tour API를 통해 얻어오게 됩니다.
이후 백그라운드에서 동작을 하면서 GPS로부터 현재 내 위치를 계속 파악하는 절차를 수행합니다.
이때 만약 내가 관광 장소에 인근에 위치해 있을 경우 사용자에게 관광지에 대한 알림 서비스를 제공해줍니다.
알림 서비스를 받고 도착한 관광지에서는 AR 기반의 게임 요소를 즐길 수 있는 기능이 구현되어 있습니다.

시스템 구현

안드로이드 스튜디오를 이용해 GPS 정보를 백그라운드에서 계속 추적하는 절차를 구현하였으며,
공공 데이터 포털의 Tour API를 통해 오늘 날짜의 관광 정보(주제, 일정, 좌표 등)를 얻어와 현재 위치에서 1km거리 반경에 관광 장소가 존재한다면 Push 알람 서비스를 보내도록 구현하였습니다.
또한, Unity 3D엔진과 Vuforia를 이용해 AR 게임 요소를 구현하여 관광에 재미를 더 했습니다.

Run TO YOU

교내 해커톤 주제 : 스마트 City 구축을 통한 사회 문제 해결

팀 주제 : 얼굴인식 API을 이용한 미아 찾기 SW


문제점 인식

1. 사람들이 밀집된 장소에서 미아가 되어버린 아이를 찾는 것은 매우 힘들다.
2. 대중들이 밀집한 장소에는 CCTV가 존재하지만 모든 장소에 배치되어이 있지 않다.
3. 미아가 된 아이를 찾기 위해서는 직접 발로 뛰거나 CCTV를 계속 주시해야 한다.
4. 결국은 미아가 되어버린 아이들이 실종 아동이 되어버린다.
이와 같은 문제를 해결하기 위해 우리 팀은 얼굴 인식 CCTV를 구현하였습니다.

시스템 흐름도

부모 혹은 관리자가 웹 사이트에 아이의 사진을 업로드 한다. 업로드된 사진은 서버를 통해 CCTV(어플)로 전달된다.
사진을 전달받은 CCTV(어플)은 얼굴인식 절차를 수행하고, 만약 동일인물로 추정되는 아이를 인식하였을 경우, 관리자에게 알림 서비스를 제공하고, 웹 사이트에 해당 카메라의 위치와 인식 시간을 전송하는 흐름을 갖는다.

시스템 구현

안드로이드 기기를 CCTV로 대체하여 안드로이드 스튜디오를 통해 현재 촬영하고 있는 영상을 사진으로 변환하는 절차와 아이의 얼굴과 사진을 비교하는 절차를 구현하기 위해 Microsoft Azure 플랫폼의 Face API를 활용하여 구현하였습니다. 웹사이트의 경우 Bootstrap 템플릿을 이용였으며 네이버 Maps API를 활용하였고, Node.js를 이용해 서버를 구축하였습니다.

참고자료 : Microsoft Azure Face API https://azure.microsoft.com/ko-kr/services/cognitive-services/face/

심사평

CCTV의 저화질 문제로 인해 실 생활에 적용될 가능성은 낮으나, 아이디어 측면에서는 굉장히 흥미로운 주제이며 상당한 구현 난이도를 보여준 것 같다.

My doctor A


1. Program Introduction

1.1 Overview

• Title : 2018 Heart-Related IoT Tracking Platform Challenge
• Date : 2018.01.08~2018.02.23(47days)
• Place : Qualcomm Institute, UCSD(University of California, San Diego!!)
• My Role : Team Leader and Android Application Development
• Participating Universities : 조선대학교, 동의대학교, 성균관대학교, 중앙대학교

1.2 Goals of Program

• To expose the students to the Health-related IoT systems.
• To explore ways to collect, transmit, and save data such as Air pollution, Heart, GPS information
• To track and visualization the locations of moving IoT sensors and data collected from them in real time.

1.3 App Client(Implemented using Android)

• User management function
   – Sign-up/sign-in/sign-out/ID cancellation
   – Authentication: Hash/token/email
• Measure its own location using GPS module
• Receive and store the following data
   – Air pollutant measurement (from UDOO board): spatial-temporary data
   – Health-related measurement (from Polar): a user-dependent data
   – GPS data
• Display its location and the data
   – Using Google map, location, and chart APIs
• Transmit and receive data to/from a Web server
• Display air quality of its neighboring area or the area requested by a user
   – Location and air quality data from more than one UDOO boards
• Display historical data by retrieving the relevant data from a Web server
   – Heart-related data / Air quality data

※ I didn't write down other parts.


2. System Introduction

Our Project Name is My Doctor A

Our project can see my Heart-related data and Air pollution data anytime, anywhere and it can project me from the dangers by connecting this information to Doctor anytime.
